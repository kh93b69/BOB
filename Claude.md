# Бот БОБ — памятка проекта

## Что это
Telegram-бот, который в общем чате напоминает ученикам о сдаче домашнего задания по расписанию.

## Зачем
Ученики (Пётр и Аник) забывают сдавать ДЗ вовремя. Бот регулярно и предсказуемо напоминает о дедлайнах, чтобы снять эту проблему.

## Пользователи
- **Ученики** — Пётр и Аник. Получают напоминания в общем чате.
- **Администратор** — преподаватель (владелец проекта). Настраивает расписание напоминаний и тексты сообщений.

## Этапы разработки
1. **MVP** — бот в общем чате, расписание в `schedule.yaml` в репо. **Без БД.** Правка расписания = git push.
2. **Админка** — админ через ЛС боту настраивает напоминания. Подключаем Supabase (без БД не обойтись: Railway filesystem ephemeral).
3. **«Нужна помощь»** — команда активирует Claude API, нейросеть помогает ученику по ДЗ.

## Стек (зафиксирован)
- **Язык**: Python 3.12
- **Бот**: aiogram 3.x (async, long-polling)
- **Планировщик**: APScheduler 3.x с `misfire_grace_time=300`
- **Расписание на MVP**: `schedule.yaml` в репо (без БД)
- **БД (этап 2+)**: Supabase (PostgreSQL) + SQLAlchemy async + asyncpg, миграции Alembic
- **Хостинг**: Railway (worker process, автодеплой из GitHub `main`)
- **Репозиторий**: GitHub — `kh93b69/BOB`
- **Нейросеть (этап 3)**: Claude API через `anthropic` SDK

## Архитектура (кратко)
Один Python-процесс на Railway: aiogram + APScheduler в общем event-loop.
БД (Supabase) — единственный источник правды о расписании: при старте и после любых изменений планировщик синхронизируется с таблицей `reminders`.

Подробности, схема БД, env-переменные, план по этапам — в [TECH_SPEC.md](TECH_SPEC.md).

## Ключевые требования
- Напоминания уходят **строго в указанное время** — это главный риск проекта.
- В админке задаются: **конкретная дата, конкретное время, конкретный текст**.
- Любая работа со временем — с явным указанием IANA-таймзоны. Проект: `Asia/Almaty` (UTC+5, Казахстан).
- Каждая отправка логируется в таблицу `send_log` — можно доказать факт отправки.
- При ошибке отправки: 3 повтора с интервалом 30 секунд + уведомление админу в ЛС.

## Принципы работы над проектом
- Идём строго по этапам: MVP → Админка → Нейросеть. Не смешиваем.
- Каждый этап должен быть полностью рабочим, прежде чем переходить к следующему.
- Точность времени — приоритет №1. Все функции, работающие со временем, покрываем тестами.
- Секреты никогда не коммитим: `.env` в `.gitignore`, значения — только в Railway env.
- БД — единственный источник правды о расписании. Планировщик никогда не хранит job мимо БД.
